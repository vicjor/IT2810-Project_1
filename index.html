About the artist

Unless you have been living under a rock for the last year, you have probably heard of the up-and-coming Trondheim artist Ragnar Rock. During 2018 Ragnar rose to fame for his now famously debut exhibition Svev over og klikk på mæ. This site serves you a two of his most popular samples from said exhibtion. The artist himself defines his art as interactive contemporary art, with inspiration from geometry. If you like what you see here, you can follow Ragnar on social meda. CSS Flexbox

CSS Flexbox and Grid are both technologies design responsive layouts in a web page, without using float and positioning for every element.

Having little to no experience on beforehand with CSS Flexbox/Grid, I had to do some research before deciding which one to use. I learned that Grid is for two-dimensional layouts and Flexbox is one-dimensional. For my project it seemed easier to just arrange elements in one direction, so I used Flexbox. Looking at code examples for both of them, Flexbox also seemed easier to use/learn. I played around, and with just a few lines of CSS i arranged my elements in the way I wanted. I decided to place my SVG and Canvas next to each other on a row with the same background styling, with a short description on top of each of them explaining how to interact with the images, and not making the layout to complicated.

SVG and Canvas

In this project it was a requirement to use both SVG and Canvas to create two different (or identical) interactive images. SVG and Canvas is two different technologies with the same purpose - to create graphics on a web page. The two of them creates graphic in different ways. Canvas uses Javascript to make drawings, and you have to define several points to create a shape. SVG uses a XML structure, so every element is available in the SVG document object model (DOM). With SVG you can easily add JavaScript event handlers to each object, which is not available with canvas for each drawn object.

Having no experience with either of them, I started playing with SVG. I used W3Schools documentation to create some simple figures like circles, rectangles, ellipses and so on. Next I tried to play with the different attributes of the HTML elements to change colors, sizes and placement. When I understood how to use SVG, I tried to add a few JavaScript functions to handle clicks and mouseovers. I started out simple with changing the colors and the size of the elements. I then decided to create a pool of five different gradient colors, and to change the color randomly to one of these five on events.

After I had basic elements in my SVG image I made a few more complex functions. I found CSS online to make the circle spin across the screen, and in addition to changing color on mouseover, I sped up the animation. I also found code online to add drop shadow to the different shapes. With my rectangle I added two event handlers, one for clicking and one for mouseover. On mouseover the rectangle will expand, and on click it will change color and return to its initial size.

To make sure I had enough variation between my SVG-elements, I made two ellipses. One static one, and another one that blends in with the background, with a stroke around to make it visible. On mouseover, the ellipse will change its location randomly to one of 25 different locations (5*5). One of these locations will make the ellipse cover the static one, blending it into the background of the SVG.

After I felt somewhat finished with my SVG image I started researching Canvas. It did not take long till I realized creating the same picture in Canvas would require a lot more effort than it did with SVG. Creating a simple square in Canvas required several lines of JavaScript, defining point to point. After I started to get the hang of the drawing properties with Canvas I tried adding some functionality. It turned out to be a difficult task to define click events on the square that I had made rather than on the Canvas itself. Apparently canvas does not support event handlers. Therefore I decided to create a different kind of image with the Canvas, with simpler functionality. On each click on the canvas, a triangle will be added to one of the sides. After four triangles is added, the next click will reset the drawing to the initial square. I concluded that if I were to recreate the same image I made with SVG, I would have to write a lot more JavaScript code, and the project would become much more time consuming than the estimated 20 hours.

jQuery

jQuery is a JavaScript library with the intention of simplifying HTML scripting. With jQuery, just like Flexbox, Canvas and SVG, I had no previous experience. Experience with JavaScript helped me learn jQuery faster. After a short tutorial on scrimba.com I quickly discovered the power using jQuery. All id’s, classes, variables and elements are easy to access and modify attributes of. This makes it incredibly simple to add functionality in response to events with the HTML-elements.

I wrote several functions for event handling in JavaScript, and made calls to them with click/mouseover using jQuery. Some of the events were one-liners, like changing size of a rectangle, and in those cases I used plain jQuery without use of self-defined functions. jQuery also made it easy to show/hide documentation on click.

Testing

Cross-browser testing is important in web development as browsers might implement different technologies, and may have different behaviour as a consequence. A consequence might be that the web page renders different in different browsers. I tested my web page in Google Chrome Version 76.0.3809.100 and in Opera Version:63.0.3368.53. All the text had the same formatting and interactions with SVG objects and the Canvas element also had the same behaviour. My web page were also responsive to change in window size in both browsers.
